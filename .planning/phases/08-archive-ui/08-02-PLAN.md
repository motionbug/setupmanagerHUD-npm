2. Add archivingIds state:
   const [archivingIds, setArchivingIds] = React.useState<Set<string>>(new Set());

3. Create handleArchive function with these parameters:
   - eventId: string
   - originalIndex: number

4. Implement handleArchive sequence:
   a. Add eventId to archivingIds Set (triggers fade transition in table)
   b. Store reference to removed event for rollback: const removedEvent = events.find(e => e.eventId === eventId);
   c. After 150ms delay (matches transition duration), remove event from local events array
   d. Fire PATCH /api/events/${encodeURIComponent(eventId)}/archive
   e. On success: remove eventId from archivingIds
   f. On failure: 
      - Remove eventId from archivingIds
      - Re-insert removedEvent at originalIndex using splice
      - Call toast.error with title "Archive failed" or "Unarchive failed" based on showArchived state
      - Description: "Could not archive this enrollment. Please try again." or "Could not restore this enrollment. Please try again."

5. Pass to EventsTable:
   - showArchived={showArchived}
   - archivingIds={archivingIds}
   - onArchive={handleArchive}

Note: Use encodeURIComponent(eventId) because eventIds contain colons (e.g., com.jamf.setupmanager.started:SERIAL:timestamp:uuid).

Note: Use functional setState updates for archivingIds to prevent race conditions when multiple rows are archived rapidly.
