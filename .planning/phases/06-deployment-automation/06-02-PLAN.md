   - Badge image: https://img.shields.io/badge/Use%20this%20template-238636?style=for-the-badge&logo=github
   - Link to: https://github.com/motionbug/setupmanagerhud-template/generate

   - Position this section BEFORE the existing "Quick Start" section
   - Header: "## Deploy via GitHub Actions (Recommended)"
   - Explain this is the browser-only path, no local tools required

3. **Document the setup steps:**
   a. Click "Use this template" to create your own repository
   b. Create D1 database in Cloudflare Dashboard:
      - Navigate to Workers & Pages > D1
      - Create database named "setupmanagerhud-events"
      - Copy the database ID
   c. Edit wrangler.toml in GitHub UI:
      - Update database_id with real ID (replace 00000000-0000-0000-0000-000000000000)
      - Commit the change
   d. Add GitHub repository secrets (Settings > Secrets and variables > Actions):
      - CLOUDFLARE_API_TOKEN: Create at dash.cloudflare.com > My Profile > API Tokens
      - CLOUDFLARE_ACCOUNT_ID: Found on any zone's Overview page in Cloudflare Dashboard
      - WEBHOOK_TOKEN: Your chosen token for webhook authentication
   e. Trigger first deploy:
      - Go to Actions tab > "Deploy to Cloudflare Workers" workflow
      - Click "Run workflow" button (workflow_dispatch)

4. **Rename existing section to "Local Development" and clarify it's optional:**
   - Change "## Quick Start" to "## Local Development (Optional)"
   - Add note that this is for developers who want to run locally before deploying

5. **Keep existing sections:** Customization, Updating, Documentation, License
